## 1. 软件简介

本工具可以将长视频文件(3-4小时)批量转换为 SRT 中文字幕，方便后续使用大模型进行数据分析。

### 核心功能

| 功能 | 说明 |
|------|------|
| 批量转换 | 支持一次上传多个视频文件 |
| 长视频处理 | 自动分段处理(每段30分钟) |
| 中文字幕 | 专为中文语音识别优化 |
| 自动切分 | SRT 文件过大时自动切分 |
| 本地运行 | 无需上传云端，保护隐私 |

### 技术特点

- **Whisper**: OpenAI 开源语音识别模型，对中文支持良好
- **CoreML 加速**: 利用 M4 芯片 Neural Engine，提速约 3-5 倍
- **FFmpeg**: 业界标准的音视频处理工具

## 4. 使用方法

### 启动软件

```bash
cd /Users/huahua-macmini/Desktop/视频转str/video-to-srt
streamlit run app.py
```

终端会显示：

```
Local URL: http://localhost:8501
Network URL: http://192.168.x.x:8501
```

浏览器会自动打开，或手动访问显示的地址。

### 操作步骤

#### 4.1 设置参数

在左侧边栏进行设置：

1. **模型选择**: 默认选择 `small`，推荐配置
2. **输出目录**: 设置 SRT 文件保存位置
3. **切分阈值**: SRT 超过此大小则自动切分(默认 10MB)

#### 4.2 上传视频

1. 点击 "拖拽或选择视频文件" 区域
2. 选择一个或多个视频文件
3. 支持的格式: MP4, AVI, MKV, MOV, WebM

#### 4.3 开始转换

1. 确认参数设置无误
2. 点击 "开始转换" 按钮
3. 观察右侧日志区域的进度

#### 4.4 获取结果

转换完成后：
- SRT 文件保存在指定的输出目录
- 如果文件过大被切分，会生成 `xxx_part1.srt`, `xxx_part2.srt` 等文件

## 6. 常见问题

### Q1: 首次运行很慢?

首次使用会下载 Whisper 模型(small 约 500MB)，请确保网络畅通。

### Q2: 处理速度很慢?

1. 确认使用的是 `small` 模型而非 `medium`
2. Mac M4 系列有 CoreML 加速，会明显更快
3. 内存建议 16GB，8GB 可能需要关闭其他应用

### Q3: 识别准确率不高?

1. 音频质量：确保视频声音清晰
2. 背景音乐：背景音乐过强可能影响识别
3. 方言：Whisper 对标准普通话识别最好

### Q4: SRT 文件乱码?

本工具生成的 SRT 使用 UTF-8 编码，用任何文本编辑器或播放器打开都应该正常。如果播放器显示乱码，请将播放器编码设置为 UTF-8。

### Q5: 如何批量处理?

直接在上传区域选择多个文件，或一次拖拽多个文件即可。

### Q6: 内存不足?

1. 关闭其他占用内存的应用
2. 使用 `tiny` 或 `base` 模型(精度较低)
3. 分批处理，不要同时处理太多视频

### Q7: 转换中断怎么办?

目前不支持断点续传，请重新开始转换。处理中的临时文件会在失败后自动清理。

## 附录: Whisper 模型对比

| 模型 | 参数量 | 内存占用 | 速度 | 中文精度 | 推荐场景 |
|------|--------|----------|------|----------|----------|
| tiny | 39M | ~1GB | 32x | 较差 | 快速测试 |
| base | 74M | ~1GB | 16x | 一般 | 低配设备 |
| **small** | 244M | ~2GB | 6x | **良好** | **日常使用** |
| medium | 769M | ~5GB | 3x | 很好 | 高精度需求 |

> 速度指相对实时音频的倍数，32x 表示 1 小时音频约 2 分钟处理完成
